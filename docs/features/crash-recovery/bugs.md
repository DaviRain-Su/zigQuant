## 潜在风险

### 风险 #1: 检查点时机

**严重性**: Medium

**描述**: 如果在检查点之后、下次检查点之前崩溃，可能丢失中间状态。

**缓解**:
- 交易后立即检查点
- 增加检查点频率
- 使用 WAL (Write-Ahead Log)

### 风险 #2: 交易所状态不一致

**严重性**: High

**描述**: 恢复时交易所状态可能已变化。

**缓解**:
- 恢复后始终同步
- 处理所有差异情况
