## 潜在风险

### 1. 时间对齐问题

**风险**: 不同数据源时间戳不一致导致事件顺序错误

**缓解措施**:
- 使用优先队列按时间排序
- 强制 UTC 时间戳
- 提供时间戳校验

### 2. 数据缺失

**风险**: CSV 数据缺失或 WebSocket 断连导致数据空洞

**缓解措施**:
- 检测数据间隔异常
- WebSocket 自动重连
- 缓存最后已知状态

### 3. 内存增长

**风险**: 长时间运行历史数据积累

**缓解措施**:
- 使用环形缓冲区
- 配置历史保留限制
- 定期清理过期数据

### 4. Code Parity 偏差

**风险**: 回测和实盘数据格式不一致

**缓解措施**:
- 统一数据标准化层
- 完整的 Code Parity 测试
- 文档明确数据格式

### 5. WebSocket 延迟

**风险**: 实盘数据延迟影响策略

**缓解措施**:
- 延迟监控和告警
- 多路数据源冗余
- 断线快速重连
